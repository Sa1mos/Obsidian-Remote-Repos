## Answers

1. Класстерный индекс - это тип индекса, который упорядочивает таблицу на диске.
   Он один, тк данные могут быть отсортированны только одним способом.
2. Не являются.
   В реляционных БД используется система связей с помощью ключей, а в документо-ориентированных - вложенные структуры.
   Хренение данных в реляционных БД реализована в виде блоков фиксированного размера в памяти, когда как документо-ориентированные БД хранят их в виде автономных документов JSON, BSON или XML.
3. `WHERE` используется для фильтрации. Он указывает, что действие должно применяться только к тем записям, которые соответсвуют заданному условию.
4. `INNER JOIN` - Только совпадающие строки из обеих таблиц
   `LEFT JOIN` - Все строки из левой + совпадения из правой (NULL если нет)
   `RIGHT JOIN` - Все строки из правой + совпадения из левой (NULL если нет)
   `FULL OUTER JOIN` - Все строки из обеих таблиц, NULL там где нет совпадений
   `CROSS JOIN` - Декартово произведение — каждая строка с каждой
   `NATURAL JOIN` - Автоматически соединяет по одноимённым столбцам
   `SELF JOIN` - Таблица соединяется сама с собой (через алиас)
   `LATERAL JOIN` - Подзапрос справа может ссылаться на столбцы из левой части
5. Иерархическая структура — это способ организации данных в виде **дерева**, где каждый элемент (узел) имеет ровно одного родителя, а корневой элемент — не имеет ни одного.
   Виды связей:
	  - Adjacency List (Список смежности)
		Самый простой способ — каждая запись хранит ссылку на родителя.
	  - Materialized Path (Путь в строке)
	    Каждая запись хранит полный путь от корня.
	  - Nested Sets (Вложенные множества)
	    Каждый узел хранит левую и правую границу своего поддерева
	  - Closure Table (Таблица замыканий)
	    Отдельная таблица хранит **все** пары предок–потомок.
6. Виды индексов:
	   - B-Tree (по умолчанию)
	     Самый распространённый. Подходит для большинства задач.
	   - Hash
	     Оптимизирован только для проверки равенства.
	   - GiST (Generalized Search Tree)
	     Обобщённое дерево поиска — фреймворк для сложных типов данных.
	   - GIN (Generalized Inverted Index)
	     Инвертированный индекс — один ключ указывает на множество строк.
	   - BRIN (Block Range Index)
	     Хранит мин/макс значения для диапазонов блоков — очень компактный.
	   - SP-GiST (Space-Partitioned GiST)
	     Разбивает пространство на непересекающиеся области.
7. Альтернативы подзапросам `JOIN`, `CTE (WITH)`, `LATERAL JOIN`, оконные функции (`OVER`).
